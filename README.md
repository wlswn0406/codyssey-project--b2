# Codyssey Team Project

최신 AI 뉴스를 자동 수집하고, 
생성형 AI로 요약한 뒤 협업 도구에 자동 저장하는 뉴스 요약 자동화 파이프라인을 팀원별로 구현함



## 미션 소개

수집부터 분석까지 자동으로 동작하는 뉴스 요약 시스템을 구축하고, 스케줄 트리거·주제 필터링·AI 요약·에러 처리까지 자동화 파이프라인의 전체 흐름을 설계하고 연결함

- **수집**: RSS 피드를 통해 최신 AI 뉴스를 자동 수집
- **요약**: 생성형 AI를 활용해 뉴스 내용을 3줄 이내로 요약
- **저장**: 노션 등 협업 도구의 데이터베이스에 자동 저장
- **자동화**: 스케줄 트리거로 매일 자동 실행, 사람 개입 없음

## 공통 요건

**주요 흐름**
```
타겟 => 뉴스 수집 => 분석(요약) => 저장 => 가공(발행)
```

### 타겟

- **주제**: AI
- **키워드 예시**: 신기술, 모델, 기업, 정책, 규제, 보안 등 (팀원별 자율 선택)

요건
- 매일 특정 시간에 자동 실행되는 스케줄 트리거 설정
- 전체 프로세스가 사람의 개입 없이 자동으로 완료
- AI 요약·저장 단계 실패 시 최대 2회 재시도

### 뉴스 수집

뉴스 소스 자율 선택 (예시)
- Google News RSS (news.google.com/rss/search?q=AI)
- TechCrunch AI (techcrunch.com/category/artificial-intelligence/feed/)
- VentureBeat AI (venturebeat.com/category/ai/feed/)
- AI News (artificialintelligence-news.com/feed/)

요건
- 링크·제목·본문 내용 포함
- 동일 기사 중복 저장 방지 키 1개 이상 적용 (GUID 또는 링크 해시)
- 수집 실패 시 재시도 없이 스킵

### 분석 및 요약

요건
- 수집된 뉴스 중 1건 선정
- 생성형 AI를 활용해 본문 내용을 3줄 이내로 요약
- 기사 1건당 요약 호출 1회 원칙 (재시도 제외)

### 저장

- 저장 도구 자율 선택 (Notion 권장, 다른 도구 사용 시 명시)

요건
- 제목·요약문·원문 링크·발행일시 속성 저장

### 에러 처리

요건
- 뉴스가 수집되지 않거나 처리 중 오류 발생 시 적절히 처리
- AI 요약·저장 단계 실패 시 최대 2회 재시도, 수집 실패는 재시도 없이 스킵

### 보안

요건
- API 키/토큰은 환경 변수 또는 비공개 저장소에 보관
- 문서·스크린샷에 API 키/토큰 노출 금지 (마스킹 처리)


## 보너스 과제

- **썸네일 이미지 자동 생성**: 이미지 생성 AI를 활용해 뉴스 주제에 맞는 썸네일 생성 후 함께 저장
- **감성 분석 태그 추가**: 뉴스 내용의 긍정/부정 감성을 분석해 태그로 분류



## 공통 제약 사항

### 도구

- 자동화 툴은 Make, Zapier, n8n 등 자유롭게 선택하되 사용 도구 명시
- 생성형 AI 연동 시 OpenAI API 또는 이에 준하는 모델 사용
- 저장 도구는 Notion 권장, 다른 도구 사용 시 명시

### 금지

- RSS 피드가 없는 사이트를 강제 크롤링해 저작권을 침해하는 방식
- 무한 루프에 빠지는 트리거 설정 (불필요한 API 호출 비용 발생)


## 최종 결과물

각 팀원은 아래 3가지를 본인 폴더에 제출

1. **자동화 워크플로우**
   - 워크플로우 구조를 보여주는 스크린샷
   - 각 단계별 역할과 연결 구조를 설명하는 문서

2. **DB 저장 결과**
   - AI가 요약한 텍스트가 포함된 뉴스 콘텐츠
   - 제목·요약문·원문 링크·발행일시가 올바른 속성에 저장된 스크린샷

3. **README**
   - 도구 선택 이유 및 필터링 기준
   - 에러 처리 정책과 선택 이유
   - 구현 과정 및 회고
